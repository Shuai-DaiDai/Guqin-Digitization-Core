# OCR 人工校对静态站点设计

## 目标

给非技术校对人员一个可直接打开的网址，用最直观的方式检查 OCR 结果：

- 左侧看原始页图和框
- 右侧看系统当前判断
- 点击框和条目时联动高亮
- 直接标记“正确 / 错误 / 跳过”
- 浏览器本地自动保存
- 随时导出结果文件

## 设计选择

这次采用“纯静态站点 + 批次数据打包”的方式，而不是先做在线写库服务。

原因：

1. `here.now` 当前提供的是静态托管，适合直接发布 HTML/CSS/JS 和图片。
2. 这个项目现在最需要的是把校对工作尽快跑起来，而不是先搭复杂后端。
3. 静态页可以先解决“谁都能打开、谁都能看、谁都能点”的问题。
4. 校对结果先导出成文件，再由我们回灌，风险更低，也更适合早期流程。

## 组成

1. `dataset-tools export-review-site`
   从现有 bundle 和批次数据导出一个完整站点目录。

2. `apps/review-studio/static/`
   静态前端模板，负责展示页图、框、当前判断和本地保存。

3. 输出站点目录
   包含：
   - `index.html`
   - `app.js`
   - `styles.css`
   - `data/review-data.json`
   - `assets/pages/*.png`

## 交互

1. 按页切换
2. 点击框选中条目
3. 点击右侧列表同步选中框
4. 标记正确 / 错误 / 跳过
5. 自动保存到浏览器本地
6. 导出 JSON / CSV 交回项目

## 后续扩展

如果之后确实需要多人在线共写，再追加真正的后端保存层即可。
但第一版先把“好用的校对界面”做出来，优先解决生产力问题。
