# OCR 运行环境准备

这份清单不是“最终生产环境规范”，而是当前第三部分从基线实验走向真训练时，最值得优先准备的一版环境建议。

## 1. 建议分成两套环境

### 本地开发机

适合做：

- 小样例调试
- 数据格式检查
- 导出训练集
- 跑启发式 baseline
- 查看 OCR bundle 和回归报告

建议配置：

- macOS 或 Linux 都可以
- Python 3.11
- 16 GB 及以上内存
- 50 GB 以上可用磁盘

### 训练机

适合做：

- 训练页级检测模型
- 做批量推理
- 跑后续 radical / component 识别实验

建议配置：

- Ubuntu 22.04 或 24.04
- NVIDIA GPU
- CUDA 12.x 对应驱动
- 24 GB 显存更稳妥
- 12 到 16 GB 显存可以先跑小模型和小批次实验
- 64 GB 内存更合适
- 500 GB 以上 SSD

## 2. 当前最需要的 Python 环境

当前仓库里第三部分已经列出的依赖在：

- `apps/ocr-engine/requirements.txt`

当前最直接的一套建议是：

- Python 3.11
- `torch`
- `torchvision`
- `ultralytics`
- `opencv-python`
- `numpy`
- `Pillow`
- `tqdm`

如果训练机使用 GPU，最关键的是先把 `torch` 和 CUDA 对齐，再装其他包。

## 3. 目录和磁盘建议

建议在训练机上单独准备这几类目录：

- 原始页图目录
- 清洗后的工作区目录
- 训练导出目录
- 模型权重目录
- 实验输出目录

建议至少预留：

- 原始与清洗数据：100 GB 起
- 训练导出与缓存：100 GB 起
- 权重和实验结果：50 GB 起

如果后面开始收集更多影印谱页，磁盘压力会上得很快，所以最好一开始就按 300 到 500 GB 可用空间来准备。

## 4. 当前阶段最值得优先准备的东西

如果你现在只想先把“能训起来”这件事准备好，优先级建议是：

1. 一台 Linux + NVIDIA 的训练机
2. Python 3.11 独立虚拟环境
3. 对应 CUDA 版本的 PyTorch
4. 一块足够大的 SSD 空间
5. 一批真实页图和已经过第二部分整理的数据

## 5. 当前不必一开始就重度准备的东西

这些以后会需要，但不是现在最卡脖子的：

- 多 GPU 分布式训练
- 复杂模型服务编排
- 在线推理服务
- 大规模数据库
- Web 标注平台自部署

## 6. 现在这套环境是为哪一步准备的

当前第三部分已经能做：

- 读取页图
- 生成 OCR bundle
- 导出裁切图
- 输出候选和回归报告
- 从第二部分 bundle 导出检测训练集

所以下一阶段真正会用到训练机的地方，主要是：

- 训练第一个正式的页级检测模型
- 跑验证集
- 批量推理并回灌到第二部分
