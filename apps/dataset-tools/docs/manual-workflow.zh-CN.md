# 手工冷启动工作流

这份说明面向真实谱页整理时的最小工作流。

## 你需要准备什么

1. `metadata.csv`
   每行对应一张页图。

2. `annotations.csv`
   每行对应一个你已经初步判断过的谱字或谱块。

3. 图片目录
   `image_file` 列要能对应到实际图片。

## 推荐顺序

1. 先运行 `validate-manual-csv`
   看有没有缺列、错图名、重复位置。

2. 再运行 `import-manual-csv`
   把人工准备的数据导入内部工作区。

3. 再运行 `process-bundle`
   自动生成摘要、草稿、复核队列和交付包。

4. 人工处理 `derived/review_queue/items.csv`
   这是最适合校对人员先看的表。

5. 如果已经有回填结果，运行 `apply-review-decisions`
   把人工确认结果重新写回事件草稿。

## 最重要的原则

- 不要求第一次就完全正确。
- 先把来源、页图、粗判断对应起来。
- 让系统把“还缺什么”自动列出来。
- 人工优先处理高优先级条目。
